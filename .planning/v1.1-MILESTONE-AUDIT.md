### Broken Connection Detail

**`_chat_async` in `cli.py` (lines 97-106)** does not guard against `StreamResult` being yielded as the final item from `stream_claude_response`. Phase 5 changed the bridge to yield `StreamResult` as the last item, but only the TUI path was updated. The `--no-tui` path passes `StreamResult` directly to `pace_characters`, which raises `TypeError: 'StreamResult' object is not iterable`.

**Impact:** Low — `--no-tui` mode is not a v1.1 requirement (CLI-01 is deferred to v1.2). The error occurs after the full response has been streamed, so users see the complete response before the crash.

**Fix:** Add `isinstance(item, StreamResult)` guard in `_chat_async`, matching the TUI pattern.
