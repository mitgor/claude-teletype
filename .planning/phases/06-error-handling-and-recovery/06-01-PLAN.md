    **src/claude_teletype/bridge.py** (update existing):

    1. Add module-level constants:
       - `READ_TIMEOUT_SECONDS = 300`
       - `POST_RESULT_TIMEOUT_SECONDS = 30`

    2. In `stream_claude_response`, replace `line = await proc.stdout.readline()` with:
       ```python
       line = await asyncio.wait_for(proc.stdout.readline(), timeout=current_timeout)
       ```
       where `current_timeout` starts at `READ_TIMEOUT_SECONDS` and switches to `POST_RESULT_TIMEOUT_SECONDS` after a result message is parsed.

    3. Add `except asyncio.TimeoutError` block in stream_claude_response that implements kill-with-timeout (ERR-04):
       ```python
       except asyncio.TimeoutError:
           # Kill subprocess with SIGTERM -> wait 5s -> SIGKILL (ERR-04)
           proc.terminate()
           try:
               await asyncio.wait_for(proc.wait(), timeout=5.0)
           except asyncio.TimeoutError:
               proc.kill()
               await proc.wait()
           yield StreamResult(
               is_error=True,
               error_message=f"Claude Code subprocess timed out (no output for {current_timeout} seconds)",
           )
       ```
       This block MUST come before any broader exception handler. The explicit terminate -> wait -> kill pattern ensures ERR-04 (zombie process cleanup) is satisfied on the timeout error path.

    4. Verify the existing `except BaseException` block still covers other error paths (non-timeout exceptions). It already calls `proc.terminate()` — confirm this is unchanged.
