  <action>
    **cli.py changes:**

    Add a `check_claude_installed()` function at module level:
    ```python
    import shutil

    CLAUDE_INSTALL_URL = "https://claude.ai/install.sh"
    CLAUDE_DOCS_URL = "https://code.claude.com/docs/en/quickstart"

    def check_claude_installed() -> None:
        if shutil.which("claude") is None:
            console.print(
                "[bold red]Claude Code CLI is not installed.[/bold red]\n\n"
                "Install it with:\n"
                f"  curl -fsSL {CLAUDE_INSTALL_URL} | bash\n\n"
                f"Or visit: {CLAUDE_DOCS_URL}",
            )
            raise typer.Exit(1)
    ```

    Call `check_claude_installed()` at the start of the `chat()` function, BEFORE the teletype branch, printer discovery, or TUI launch. This catches missing CLI early with a clear message instead of a subprocess FileNotFoundError deep in the bridge.

    **tui.py changes:**

    Replace the existing `stream_response` worker method with a version that:

    1. Adds retry constants: `MAX_RETRIES = 3`, `BASE_DELAY = 1.0` at module level.

    2. Wraps the streaming loop in a retry `while` loop (max MAX_RETRIES iterations for retryable errors).

    3. After receiving a `StreamResult` with `is_error=True`:
       a. Import and call `classify_error(item.error_message)` from `claude_teletype.errors`
       b. Import `is_retryable` and `ERROR_MESSAGES` from `claude_teletype.errors`
       c. If the error is retryable AND retries < MAX_RETRIES:
          - Increment retry counter
          - Calculate delay: `BASE_DELAY * (2 ** (retries - 1)) + random.uniform(0, 1)`
          - Write `f"\n[{ERROR_MESSAGES[category]} Retrying in {delay:.0f}s... (attempt {retries}/{MAX_RETRIES})]\n"` to log
          - `await asyncio.sleep(delay)`
          - `break` out of the inner `async for` to retry
       d. If NOT retryable OR max retries reached:
          - Write `f"\n[{ERROR_MESSAGES[category]}]\n"` to log
          - Do NOT retry (fall through to return)

    4. Handle session recovery (ERR-06 enhancement): When `is_error` and `self._session_id is not None`:
       - Classify the error
       - Reset `self._session_id = None` (critical: must happen BEFORE any retry)
       - Show the classified message: `f"\n[{ERROR_MESSAGES.get(category, ERROR_MESSAGES[ErrorCategory.SESSION_CORRUPT])}]\n"`
       - If also retryable, proceed with retry logic (but with session_id now None, so retry starts fresh)

    5. Replace the generic `except Exception as exc: log.write_line(f"\n[Error: {exc}]")` with a classified handler that tries to extract useful info from the exception.

    6. On retry, do NOT clear already-written output. Only retry if no text chunks were yielded before the error (track with a `has_text = False` flag set to `True` on first pace_characters call). If text was already streamed, do NOT retry (partial response duplication issue from research Pitfall 4). Instead show the error message and let user resubmit.

    **tests/test_tui.py changes:**

    Add tests:
    - Test that check_claude_installed raises typer.Exit when shutil.which returns None (mock shutil.which)
    - Test that check_claude_installed succeeds when shutil.which returns a path (mock shutil.which)

    Do NOT attempt to write complex async TUI worker retry tests -- the retry logic is tightly coupled to the Textual worker lifecycle which is difficult to unit test. The error classification logic is already thoroughly tested in test_errors.py from Plan 01. The TUI integration will be verified via the end-to-end verification commands below.
