**tui.py changes:**

1. Add tracking attributes to `__init__`:
   - `backend_name: str = "claude-cli"` parameter → `self._backend_name = backend_name`
   - `model_config: str = ""` parameter → `self._model_config = model_config`
   - `profile_name: str = "generic"` parameter → `self._profile_name = profile_name`
   - `all_profiles: dict | None = None` parameter → `self._all_profiles = all_profiles or {}`

2. Add keybinding to BINDINGS list — place between ctrl+t and escape:
   `Binding("ctrl+comma", "open_settings", "Settings")`
   Per research (Open Question 1), ctrl+s is avoided because it triggers XOFF terminal freeze in many terminal emulators. ctrl+comma is the standard "settings" shortcut (VS Code, Sublime Text, Discord). If Textual's key parser does not accept "ctrl+comma" (verify step will catch this), fall back to `Binding("f2", "open_settings", "Settings")`. Do NOT use ctrl+s.

3. Add `action_open_settings` method:
   - Lazy import `SettingsScreen` from `claude_teletype.settings_screen`
   - Call `self.push_screen(SettingsScreen(...), callback=self._apply_settings)` passing all current values:
     - `current_delay=self.base_delay_ms`
     - `current_no_audio=self.no_audio`
     - `current_backend=self._backend_name`
     - `current_model=self._model_config`
     - `current_profile=self._profile_name`
     - `available_profiles=sorted(self._all_profiles.keys()) if self._all_profiles else ["generic"]`

4. Add `_apply_settings(self, result: dict | None)` method:
   - If result is None, return (cancelled)
   - Set `self.base_delay_ms = result["delay"]`
   - Set `self.no_audio = result["no_audio"]`
   - If backend or model changed (`result["backend"] != self._backend_name or result["model"] != self._model_config`):
     - Lazy import `BackendError, create_backend` from `claude_teletype.backends`
     - try: create new backend with `create_backend(backend=result["backend"], model=result["model"] or None)`, call `.validate()`, replace `self._backend`, update `self._backend_name` and `self._model_config`
     - except BackendError as e: `self.notify(str(e), severity="error")` — keeps old backend
   - If profile changed (`result["profile"] != self._profile_name`):
     - Update `self._profile_name = result["profile"]`
     - If `self.printer` is not None and `hasattr(self.printer, "_profile")` (i.e., it is a `ProfilePrinterDriver`):
       - Look up the new profile object: `new_profile = self._all_profiles.get(result["profile"])`
       - If found: set `self.printer._profile = new_profile` and reset `self.printer._initialized = False` so the new profile's init sequence is sent on the next write
       - This takes effect immediately: `ProfilePrinterDriver.write()` reads `self._profile` at write-time for CRLF handling, reinit sequences, etc.

**cli.py changes:**

In the `main()` function, where `TeletypeApp` is instantiated (the `else` branch around line 437), add keyword arguments:

- `all_profiles=all_profiles` (the merged dict of BUILTIN_PROFILES + custom_profiles_dict already computed above)

This passes the startup values so the settings modal can display them and detect changes.
