   ```
   "sounddevice>=0.5.0",
   "numpy>=1.26.0",
   ```
   Run `uv sync` to install.

2. Create `src/claude_teletype/audio.py` with `make_bell_output()`:
   - Lazy-import sounddevice and numpy INSIDE the function (not at module level) for graceful degradation.
   - Generate bell tone in-memory: 880 Hz sine wave, 150ms duration, 44100 sample rate, exponential decay (factor 20), cast to float32.
   - Return a `Callable[[str], None]` that calls `sd.play(bell, samplerate=sr)` when char == "\n" and does nothing for other characters.
   - Wrap the entire body in try/except (ImportError, OSError) -- on failure, return a no-op lambda that accepts char and does nothing. This handles missing PortAudio gracefully.
   - Type annotation: `def make_bell_output() -> Callable[[str], None]`
   - Use `from collections.abc import Callable` for the type hint.

3. Create `tests/test_audio.py` with tests:
   - **test_make_bell_output_returns_callable**: Verify make_bell_output() returns a callable.
   - **test_bell_output_accepts_non_newline_chars**: Call the returned callable with "a", "b", " " -- should not raise.
   - **test_bell_output_accepts_newline**: Call with "\n" -- should not raise (may or may not actually play sound in test environment).
   - **test_bell_output_graceful_degradation**: Monkeypatch/mock the sounddevice import to raise ImportError inside make_bell_output. Verify it returns a callable (no-op) and calling it with "\n" does not raise.
   - For the degradation test: use `unittest.mock.patch.dict('sys.modules', {'sounddevice': None})` to simulate missing sounddevice, then re-import or call the factory.
   - Actually simpler: since make_bell_output does lazy import, mock `builtins.__import__` to raise ImportError for 'sounddevice', then call make_bell_output() and verify the returned callable works as a no-op.
   - Alternative approach (preferred): test by patching `sounddevice` to None in sys.modules won't work reliably since numpy is also imported. Instead, just verify the return type is callable and that calling it with various chars doesn't raise. The graceful degradation is structural (try/except in the source).

4. Run `uv run pytest tests/test_audio.py -v` -- all tests must pass.
5. Run `uv run ruff check src/claude_teletype/audio.py tests/test_audio.py` -- must be clean.

<task type="auto">
  <name>Task 2: Create transcript writer module with tests</name>
  <files>src/claude_teletype/transcript.py, tests/test_transcript.py</files>
  <action>
1. Create `src/claude_teletype/transcript.py` with `make_transcript_output()`:
   - Signature: `def make_transcript_output(transcript_dir: Path | None = None) -> tuple[Callable[[str], None], Callable[[], None]]`
   - Returns a `(write_fn, close_fn)` tuple.
   - If transcript_dir is None, default to `Path.cwd() / "transcripts"`.
   - Create the directory: `transcript_dir.mkdir(parents=True, exist_ok=True)`.
   - Generate timestamped filename: `datetime.now().strftime("%Y%m%d-%H%M%S")` -> `transcript-{timestamp}.txt`.
   - Open file: `fh = open(filepath, "a", encoding="utf-8")`.
   - `write_fn(char: str)`: writes char to fh, flushes on newline (`if char == "\n": fh.flush()`).
   - `close_fn()`: if not fh.closed, flush and close.
   - Imports: `from datetime import datetime`, `from pathlib import Path`, `from collections.abc import Callable`.

2. Create `tests/test_transcript.py` with tests:
   - **test_make_transcript_output_returns_tuple**: Verify returns a 2-tuple of callables. Use `tmp_path` fixture for transcript_dir.
   - **test_transcript_writes_characters**: Create output with tmp_path, call write_fn with "H", "i", "\n". Read the transcript file -- contents should be "Hi\n".
   - **test_transcript_flushes_on_newline**: Write chars including "\n", then read file immediately (without closing) -- content through the newline should be present on disk.
   - **test_transcript_close_flushes_remaining**: Write chars without trailing newline, call close_fn, read file -- all chars should be present.
   - **test_transcript_creates_directory**: Pass a non-existent subdirectory in tmp_path as transcript_dir. Verify directory was created and file exists.
   - **test_transcript_filename_format**: Verify the created file matches pattern `transcript-*.txt` using glob on the transcript_dir.
   - **test_close_idempotent**: Call close_fn twice -- should not raise on second call.
   - Use pytest's `tmp_path` fixture for all tests to avoid polluting the filesystem.

3. Run `uv run pytest tests/test_transcript.py -v` -- all tests must pass.
4. Run `uv run ruff check src/claude_teletype/transcript.py tests/test_transcript.py` -- must be clean.
  </action>
