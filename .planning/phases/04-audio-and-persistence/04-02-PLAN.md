**CLI changes (cli.py):**

1. Add two new CLI flags to the `chat` command:
   - `--no-audio` (bool, default False): "Disable bell sound on line breaks"
   - `--transcript-dir` (str, default None): "Directory for transcript files (default: ./transcripts)"

2. Pass `no_audio` and `transcript_dir` through to both TUI and no-TUI branches.

3. In `_chat_async()`, add `no_audio: bool = False` and `transcript_dir: str | None = None` parameters. Before building destinations:
   - Lazy import: `from claude_teletype.audio import make_bell_output`
   - Lazy import: `from claude_teletype.transcript import make_transcript_output`
   - If not no_audio: `destinations.append(make_bell_output())`
   - Create transcript: `transcript_write, transcript_close = make_transcript_output(Path(transcript_dir) if transcript_dir else None)`
   - Append: `destinations.append(transcript_write)`
   - Write user prompt header to transcript before streaming: `transcript_write` receives chars through output_fn, but the user prompt is written by `_chat_async` via `print()` which goes to stdout only. To capture user input in transcript, write `f"\n> {prompt}\n\n"` char-by-char to transcript_write BEFORE the streaming loop. Actually simpler: just call `for ch in f"\n> {prompt}\n\n": transcript_write(ch)` before the streaming loop.
   - In the finally block: call `transcript_close()` after printer cleanup.

4. In the TUI branch of `chat()`, pass the new flags to TeletypeApp:
   - `TeletypeApp(base_delay_ms=delay, printer=printer, no_audio=no_audio, transcript_dir=transcript_dir)`

**TUI changes (tui.py):**

1. Update `TeletypeApp.__init__()` to accept `no_audio: bool = False` and `transcript_dir: str | None = None` parameters. Store as `self.no_audio` and `self.transcript_dir`.

2. Add instance variable `self._transcript_close: Callable[[], None] | None = None` for cleanup tracking.

3. In `stream_response()` worker, after building the printer destination:
   - Lazy import: `from claude_teletype.audio import make_bell_output`
   - Lazy import: `from claude_teletype.transcript import make_transcript_output`
   - If not self.no_audio: `destinations.append(make_bell_output())`
   - Create transcript once per response is WRONG -- we want one transcript per session, not per response. Store the transcript callable on the app instance.

   **Revised approach for TUI transcript:** Initialize transcript in `on_mount()` instead of per-response:
   - In `on_mount()`, lazy import make_transcript_output, create (write_fn, close_fn), store as `self._transcript_write` and `self._transcript_close`.
   - In `stream_response()`, add `self._transcript_write` to destinations if it exists.
   - The user prompt header is already written to log via `log.write(f"\n> {prompt}\n\n")` in `on_input_submitted`. Add a matching write to transcript: `if self._transcript_write: for ch in f"\n> {prompt}\n\n": self._transcript_write(ch)` in `on_input_submitted` after the log.write.

4. In `on_unmount()`, after printer cleanup: `if self._transcript_close: self._transcript_close()`.

5. Import Path at module level is fine since pathlib is stdlib and already available. But keep audio/transcript imports lazy inside the methods.

6. Run `uv run pytest -v` to verify all tests pass.
7. Run `uv run ruff check src/ tests/` for lint clean.
