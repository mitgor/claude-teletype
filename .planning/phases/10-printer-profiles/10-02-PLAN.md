  <name>Task 1: Replace JukiPrinterDriver with ProfilePrinterDriver and update discover_printer()</name>
  <files>src/claude_teletype/printer.py</files>
  <action>
  In printer.py:

  1. **Add ProfilePrinterDriver class** (replaces JukiPrinterDriver):
     - Constructor takes `inner: PrinterDriver` and `profile: PrinterProfile` (import from profiles module)
     - `_send_raw(data: bytes)` sends byte-by-byte through inner driver (same as JukiPrinterDriver)
     - `_ensure_init()` sends profile.init_sequence + profile.line_spacing + profile.char_pitch on first write
     - `is_connected` property delegates to inner
     - `write(char)` handles newlines: if profile.crlf, send CR before LF. If profile.reinit_on_newline, send profile.reinit_sequence after newline. Otherwise, pass through to inner.
     - `close()` sends formfeed if profile.formfeed_on_close, then reset_sequence if present, then inner.close()

  2. **Keep JukiPrinterDriver as deprecated alias**: Do NOT delete JukiPrinterDriver yet. Add a comment `# Deprecated: use ProfilePrinterDriver with get_profile("juki")`. Change its body to inherit from or wrap ProfilePrinterDriver with the juki profile. Simplest approach: make it a thin subclass that calls `super().__init__(inner, get_profile("juki"))`.

  3. **Update discover_printer() signature**: Change from `discover_printer(device_override, juki)` to `discover_printer(device_override, profile=None)` where profile is `PrinterProfile | None`. When profile is not None and not generic (has any init/reset/crlf behavior), wrap the discovered driver in ProfilePrinterDriver. Keep the same discovery priority (device override > USB > CUPS > Linux devs > Null). The `juki` parameter should still be accepted for backward compat but marked as deprecated -- if `juki=True` and no profile passed, use `get_profile("juki")`.

  4. **Update make_printer_output**: Update the A4_COLUMNS reference to use profile.columns if a profile is available, or keep 80 as default. This may require make_printer_output to accept an optional columns parameter. Keep it simple: add `columns: int = A4_COLUMNS` parameter.

  5. **Import**: Add `from claude_teletype.profiles import PrinterProfile, get_profile` at the top.
  </action>

<task type="auto">
  <name>Task 2: Extend config and CLI with --printer flag, deprecation, and custom profile support</name>
  <files>src/claude_teletype/config.py, src/claude_teletype/cli.py, src/claude_teletype/teletype.py</files>
  <action>
  **config.py changes:**

  1. Add `printer_profile: str = "generic"` field to TeletypeConfig (keep `juki: bool = False` for backward compat, will be removed later).
  2. Update DEFAULT_CONFIG_TEMPLATE: Replace `juki = false` with `profile = "generic"` in [printer] section. Add a commented-out example of [printer.profiles.custom] section showing hex-encoded fields.
  3. Update `load_config()`: Before flattening TOML sections, extract custom profiles from `raw.get("printer", {}).get("profiles", {})` and store them separately. Return them alongside the config. Two options: (a) return a tuple `(TeletypeConfig, dict)`, or (b) add a `_custom_profiles` field. Option (a) is cleaner but breaks the API. Option (b) is simpler for integration. Choose (b): add a non-TOML `custom_profiles: dict = field(default_factory=dict)` that stores raw TOML profile dicts. Load it in load_config before flattening. When flattening, skip the `profiles` nested dict (it would break flat field mapping).

  **cli.py changes:**

  1. Add `--printer` / `-p` option: `printer: str = typer.Option(None, "--printer", "-p", help="Printer profile name (e.g., juki, escp, pcl)")`.
  2. Keep `--juki` flag but add deprecation: When juki=True, emit `typer.echo("Warning: --juki is deprecated, use --printer juki", err=True)` and set effective printer to "juki".
  3. Profile resolution logic in main():
     - If `--printer` flag set: use that name
     - Elif `--juki` flag set: use "juki" (with deprecation warning)
     - Elif config.printer_profile != "generic": use config.printer_profile
     - Elif config.juki: use "juki" (backward compat for old config files)
     - Else: try auto_detect_profile() from profiles module
     - Else: use "generic"
  4. Resolve the profile name to a PrinterProfile using get_profile() and load_custom_profiles(). The custom profiles come from config.custom_profiles. Merge built-in + custom, then look up name. If name not found, print error and exit.
  5. Pass the resolved profile to discover_printer(profile=...) instead of juki=bool.
  6. Update `config show` to display `printer_profile = {config.printer_profile}` instead of (or in addition to) `juki = {config.juki}`.
  7. Update teletype mode: Pass the resolved profile to run_teletype instead of juki boolean.

  **teletype.py changes:**

  1. Change `run_teletype(driver, juki=False)` to `run_teletype(driver, profile=None)` where profile is `PrinterProfile | None`.
  2. If profile is not None: send profile.init_sequence + profile.line_spacing + profile.char_pitch as init (single write for USB bulk transfer). Use profile.crlf for newline strategy.
  3. If profile is None: behave as generic (LF only, no init codes) -- same as current default.
  4. Update the docstring and imports. Import PrinterProfile from profiles.
  5. On close: send formfeed if profile.formfeed_on_close (or always, matching current behavior). Send profile.reset_sequence if present.
  </action>
