<feature>
  <name>PrinterProfile dataclass and profile registry</name>
  <files>src/claude_teletype/profiles.py, tests/test_profiles.py</files>
  <behavior>
    PrinterProfile is a frozen dataclass with fields:
    - name: str
    - description: str = ""
    - init_sequence: bytes = b""
    - reset_sequence: bytes = b""
    - line_spacing: bytes = b""
    - char_pitch: bytes = b""
    - crlf: bool = False
    - reinit_on_newline: bool = False
    - reinit_sequence: bytes = b""
    - formfeed_on_close: bool = True
    - usb_vendor_id: int | None = None
    - usb_product_id: int | None = None
    - columns: int = 80

    BUILTIN_PROFILES is a dict[str, PrinterProfile] with 5 entries:
    - "generic": no ESC codes, LF-only
    - "juki": ESC SUB I init, RS 9 line spacing, ESC Q fixed pitch, CR+LF, reinit on newline
    - "escp": ESC @ init/reset, ESC 2 line spacing, ESC P pitch, Epson VID 0x04B8
    - "ppds": ESC @ init/reset, ESC 2 line spacing, DC2 pitch
    - "pcl": ESC E init/reset, &l6D line spacing, (s10H pitch, HP VID 0x03F0

    get_profile(name) -> PrinterProfile:
    - Case-insensitive lookup
    - get_profile("juki") -> BUILTIN_PROFILES["juki"]
    - get_profile("JUKI") -> BUILTIN_PROFILES["juki"]
    - get_profile("unknown") -> raises ValueError with available names

    load_custom_profiles(raw_toml: dict) -> dict[str, PrinterProfile]:
    - Parses [printer.profiles.*] tables from raw TOML dict
    - Hex-encoded strings converted via bytes.fromhex()
    - load_custom_profiles({"printer": {"profiles": {"my-printer": {"init": "1b40", "crlf": true}}}})
      -> {"my-printer": PrinterProfile(name="my-printer", init_sequence=b"\x1b@", crlf=True)}
    - USB VID/PID parsed as hex strings: "04b8" -> 0x04B8
    - Missing keys get defaults (empty bytes, False, etc.)
    - Empty/missing profiles section -> empty dict

    auto_detect_profile() -> PrinterProfile | None:
    - Enumerates USB devices via pyusb, filters to printer class (interface class 7)
    - Matches VID:PID against built-in profiles (+ optional custom profiles)
    - Returns matching profile or None
    - Returns None if pyusb not installed (ImportError)
    - Returns None if no USB backend (NoBackendError)
    - VID-only match when profile has usb_product_id=None
    - Exact VID+PID match takes priority over VID-only match

    Cases for tests:
    - get_profile("generic") -> PrinterProfile with empty bytes
    - get_profile("juki") -> correct init_sequence b"\x1b\x1aI"
    - get_profile("escp") -> correct usb_vendor_id 0x04B8
    - get_profile("JUKI") -> same as "juki" (case insensitive)
    - get_profile("nonexistent") -> ValueError
    - BUILTIN_PROFILES has exactly 5 entries
    - load_custom_profiles with valid hex -> correct bytes
    - load_custom_profiles with usb_vendor_id hex -> correct int
    - load_custom_profiles with empty dict -> empty dict
    - load_custom_profiles with missing optional fields -> defaults
    - auto_detect_profile with no pyusb -> None
    - auto_detect_profile with matching VID:PID -> correct profile
    - auto_detect_profile with no matching device -> None
    - auto_detect_profile with VID-only match -> correct profile
    - PrinterProfile is frozen (immutable)
  </behavior>
