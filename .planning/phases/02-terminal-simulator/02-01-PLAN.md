<task type="auto">
  <name>Task 2: Create Textual split-screen TUI application with tests</name>
  <files>src/claude_teletype/tui.py, tests/test_tui.py</files>
  <action>
1. Create `src/claude_teletype/tui.py` with class `TeletypeApp(App)`:

   **Constructor:** Accept `base_delay_ms: float = 75.0` parameter. Store as `self.base_delay_ms`.

   **CSS:** Split-screen layout:
   ```
   #output { height: 1fr; }
   #prompt { dock: bottom; }
   ```

   **compose():** Yield `Header()`, `Log(id="output", auto_scroll=True)`, `Input(id="prompt", placeholder="Type a prompt and press Enter...")`, `Footer()`.

   **TITLE:** `"Claude Teletype"`

   **BINDINGS:** Add `Binding("ctrl+d", "quit", "Quit")` so Footer shows the exit key.

   **on_input_submitted(event: Input.Submitted):**
   - Get prompt from `event.value`. If empty/whitespace, return early (do nothing).
   - Clear the input: `event.input.clear()`
   - Write the prompt to the log with a prefix: `log.write_line(f"\n> {prompt}\n")`
   - Update input placeholder to "Thinking..." to indicate processing.
   - Call `self.stream_response(prompt)` to start the background worker.

   **stream_response(prompt: str):** Decorated with `@work(exclusive=True)`.
   - Import `stream_claude_response` from bridge and `pace_characters` from pacer inside the method (lazy import avoids circular issues and keeps bridge/pacer optional for testing).
   - Import `make_output_fn` from output.
   - Get log widget: `self.query_one("#output", Log)`
   - Create output function: `output_fn = make_output_fn(log.write)`
   - Iterate `async for chunk in stream_claude_response(prompt):`
     - `await pace_characters(chunk, base_delay_ms=self.base_delay_ms, output_fn=output_fn)`
   - After loop completes, write a trailing newline: `log.write_line("")`
   - Reset input placeholder: `self.query_one("#prompt", Input).placeholder = "Type a prompt and press Enter..."`
   - Wrap the streaming loop in try/except for `CancelledError` (from `asyncio`) -- on cancel, write "[Cancelled]" to log and reset placeholder.
   - Also catch generic `Exception` to write error to log and reset placeholder (prevents silent failures).

2. Create `tests/test_tui.py` with Textual app tests using `run_test()`:

   **test_layout_has_log_and_input():**
   - Create `TeletypeApp(base_delay_ms=0)`, `async with app.run_test() as pilot:`
   - Assert `app.query_one("#output", Log)` exists
   - Assert `app.query_one("#prompt", Input)` exists
   - Assert Header and Footer are present

   **test_app_title():**
   - Assert `app.title == "Claude Teletype"`

   **test_enter_clears_input():**
   - Type text via `await pilot.press(*"hello")`
   - Press enter: `await pilot.press("enter")`
   - Assert `app.query_one("#prompt", Input).value == ""`

   **test_prompt_echoed_to_log():**
   - Type and submit "test prompt"
   - `await pilot.pause()` to let worker start
   - Check that `"> test prompt"` appears in the log lines (use `str(log.lines)` or iterate log content)

   **test_empty_input_not_submitted():**
   - Press enter with empty input
   - Assert no text was written to the log (log lines unchanged)

   **test_custom_delay_stored():**
   - `app = TeletypeApp(base_delay_ms=42.0)`
   - Assert `app.base_delay_ms == 42.0`

   Note: Do NOT test actual Claude streaming in unit tests (requires subprocess). The stream_response worker uses lazy imports so TUI tests work without mocking the bridge. Tests should focus on widget layout, input handling, and prompt echoing.
