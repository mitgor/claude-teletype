  <name>Task 1: Implement CLI with Typer, thinking indicator, and async bridge</name>
  <files>

  <action>
    Create the CLI entry point that wires together bridge and pacer with a thinking indicator.

    **src/claude_teletype/cli.py** must contain:

    1. A Typer app instance: `app = typer.Typer()`

    2. A Rich Console instance: `console = Console()`

    3. An async function `_chat_async(prompt: str, base_delay_ms: float) -> None`:
       - Sets `first_token = True`
       - Opens `console.status("[bold cyan]Thinking...", spinner="dots")` as context manager
       - Iterates with `async for text_chunk in stream_claude_response(prompt):`
         - On first chunk: calls `status.stop()`, sets `first_token = False`
         - Calls `await pace_characters(text_chunk, base_delay_ms=base_delay_ms)` for each chunk
       - After the loop: prints a final newline with `print()`
       - If `first_token` is still True after loop (no text received): prints an error message "No response received from Claude."
       - Wraps the entire body in try/except KeyboardInterrupt that prints "\n[Interrupted]" and returns cleanly

    4. A Typer command function `chat(...)`:
       - `prompt: str = typer.Argument(..., help="Prompt to send to Claude")`
       - `delay: float = typer.Option(75.0, "--delay", "-d", help="Base delay between characters in milliseconds (50-100 recommended)")`
       - Body: `asyncio.run(_chat_async(prompt, delay))`

    5. Import from bridge: `from claude_teletype.bridge import stream_claude_response`
    6. Import from pacer: `from claude_teletype.pacer import pace_characters`

    After creating cli.py, verify the entry point works:
    - `uv run claude-teletype --help` should show the command with prompt argument and --delay option
    - `uv run python -m claude_teletype --help` should produce the same output (via __main__.py)
