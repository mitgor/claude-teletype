# Phase 19 — UI Design Contract

> Visual and interaction contract for the Printer Setup Screen. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Textual TUI -- not a web project) |
| Preset | not applicable |
| Component library | Textual 7.5.0 built-in widgets |
| Icon library | none (terminal text characters only) |
| Font | Terminal default (monospace, determined by user's terminal emulator) |

**Note:** This project is a Python Textual terminal UI. All measurements are in terminal cells, not pixels. Colors use Textual CSS variables mapped to the terminal's color palette.

---

## Spacing Scale

Declared values in terminal cell units (Textual `padding` / `margin`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 0 | Tight inline grouping (radio buttons within RadioSet) |
| sm | 1 | Compact element spacing (label to widget, button gaps) |
| md | 2 | Default section padding (container padding horizontal) |
| lg | 3 | Between major sections (device list to connection method) |
| xl | 4 | Not used in this phase |

Exceptions: none. All spacing uses Textual's cell-based `padding` and `margin` properties.

---

## Typography

Terminal UI has no font size control. Typography is expressed through Textual text styles:

| Role | Style | Usage |
|------|-------|-------|
| Body | plain | Device list items, diagnostic log entries |
| Label | `text-style: bold` | Section labels ("Discovered Devices:", "Connection Method:", "Printer Profile:", "Diagnostics:") |
| Title | `text-style: bold` | Screen title "Printer Setup" |
| Muted | `color: $text-muted` | Disabled states, secondary info (VID:PID in device list) |

---

## Color

Colors use Textual CSS design tokens (resolved by terminal theme):

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `$background` | Screen background, main container |
| Secondary (30%) | `$surface` | Diagnostics log border, status areas |
| Accent (10%) | `$primary` | "Connect" button only (variant="primary") |
| Warning | `$warning` | "Install USB Support" button only (variant="warning") |
| Muted | `$text-muted` | Disabled device entries, VID:PID annotations |

Accent reserved for: the "Connect" button. No other element uses `$primary` variant.

Warning reserved for: the "Install USB Support" button. Communicates optional-but-notable action without implying destruction.

## Interaction Contract

### Device Selection Flow

1. Screen mounts with `DiscoveryResult` data pre-loaded
2. `OptionList` is populated with formatted device strings:
   - USB devices: `"{product_name} (USB {vendor_id:04x}:{product_id:04x})"` or `"USB Device ({vendor_id:04x}:{product_id:04x})"` if no product name
   - CUPS printers: `"{name} (CUPS{': ' + model if model else ''})"` 
3. User highlights a device; `OptionList.OptionSelected` fires
4. On USB device selection: enable both radio buttons, auto-select "USB Direct"
5. On CUPS device selection: auto-select "CUPS Queue", disable "USB Direct" radio
6. Profile `Select` auto-populates from `auto_detect_profile()` when VID:PID matches a known device; falls back to "generic"

### Connection Method Toggle

| State | USB Direct Radio | CUPS Queue Radio |
|-------|-----------------|-----------------|
| USB device selected, pyusb available | enabled, selected | enabled |
| USB device selected, pyusb NOT available | disabled | enabled, selected |
| CUPS printer selected | disabled | selected |
| No device selected | disabled | disabled |
| macOS detected | enabled | enabled, selected (default) |

### Profile Assignment

- `Select[str]` populated with sorted keys from `all_profiles` dict (builtin + custom)
- When user selects a USB device with a matching VID:PID, auto-set profile via `auto_detect_profile()`
- When user selects a CUPS printer, default to "generic" (no VID:PID available)
- User can always manually override the auto-suggested profile

### Install USB Support Flow

| State | Install Button | Spinner | Diagnostics Log |
|-------|---------------|---------|-----------------|
| pyusb available | hidden (display: none on `#install-row`) | hidden | -- |
| pyusb missing, idle | visible, enabled, variant="warning" | hidden | "pyusb not installed -- USB detection unavailable" |
| pyusb installing | visible, disabled | visible | "Installing pyusb via uv sync --extra usb..." |
| Install succeeded | hidden | hidden | "pyusb installed successfully." then "USB support activated. Re-scanning devices..." |
| Install failed | visible, enabled | hidden | "Install failed: {stderr}" |

### Button States

| Button | Default State | Disabled When |
|--------|--------------|---------------|
| Connect | enabled if device selected | no device highlighted in OptionList |
| Skip (Simulator) | always enabled | never |
| Install USB Support | enabled when pyusb missing | during install; hidden when pyusb available |

### Keyboard Bindings

| Key | Action | Shown in Footer |
|-----|--------|----------------|
| Escape | Skip (dismiss with None) | yes |
| Enter | Activate focused button / confirm OptionList selection | implicit (Textual default) |
| Up/Down | Navigate OptionList, RadioSet | implicit (Textual default) |
| Tab | Move focus between widgets | implicit (Textual default) |

## State Machine

### Screen Dismiss Results

| User Action | Dismiss Value |
|-------------|--------------|
| Select device + Connect | `PrinterSelection(connection_type="usb"\|"cups", device_index=N, cups_printer_name=str, profile_name=str)` |
| Skip (Simulator) | `None` |
| Escape key | `None` |

### Data Flow

```
cli.py: discover_all() -> DiscoveryResult
  |
  v
TeletypeApp(discovery=DiscoveryResult)
  |
  v (on_mount, call_after_refresh)
PrinterSetupScreen(discovery, all_profiles)
  |
  v (dismiss)
PrinterSelection | None -> _handle_setup_result()
  |
  v
create_driver_for_selection() -> PrinterDriver
```
