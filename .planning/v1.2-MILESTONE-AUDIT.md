## Integration Observations (Non-Blocking)

### 1. IBM profile named "ppds" (PRNT-02)
The built-in IBM PPDS profile is keyed as `ppds` in `BUILTIN_PROFILES`. Running `--printer ibm` shows an error with available profiles. The profile works correctly under `--printer ppds`. Discoverability issue only.

### 2. config show shows file+env only (CFG-04)
The `config show` subcommand calls `load_config()` + `apply_env_overrides()` but not `merge_cli_flags()`. Typer routes subcommands separately from main() — CLI flags cannot be passed to the show subcommand. Architectural constraint, not a code gap.

### 3. system_prompt silent drop for claude-cli backend
When `system_prompt` is set in TOML and `backend = "claude-cli"`, the system_prompt is passed to `create_backend()` but not forwarded to `ClaudeCliBackend`. Claude CLI manages its own system prompt. Documented in config template comments.

### 4. Backend hot-swap loses session_id
Switching backends via settings modal creates a new backend without `session_id`. A claude-cli → openai → claude-cli round-trip starts a fresh session. Documented design choice for v1.2.
